TP 1 – Maquettage HTML & CSS

Reproduire fidèlement des maquettes en HTML/CSS et respecter une charte graphique

Vue d’ensemble du TP

Le TP 1 est un premier travail de maquettage : il s’agit de reproduire très précisément des captures de pages web en utilisant uniquement HTML et CSS.

Ce TP permet de se familiariser avec la structure d’une page, les balises de base, la feuille de styles séparée et le respect strict d’une maquette fournie par l’enseignant.

Connaissances techniques mobilisées

  • Structure de base d’un document HTML5 : <!DOCTYPE html>, <html>, <head>, <body>.
  • Balises de contenu : titres (h1 à h3), paragraphes, listes, liens, images.
  • Séparation du fond et de la forme : fichier HTML pour le contenu, fichier CSS externe pour la présentation.
  • Utilisation des sélecteurs CSS (élément, classe, id) pour cibler les différentes zones de la page.
  • Gestion des marges, paddings, largeurs et alignements pour coller à la maquette.
  • Utilisation de couleurs, polices et arrière‑plans (dont des images) à l’aide de propriétés CSS.

Exercices réalisés

Exercice 1 – Reproduction d’une première page

Objectif : Créer une page HTML simple reproduisant une capture d’écran fournie (titres, paragraphes, liens, mise en forme de base).

Points clés : bonne hiérarchie des balises (un seul h1 principal), respect des sauts de ligne, alignements, et utilisation des styles CSS pour les couleurs et les polices.

Difficulté ressentie : faible à moyenne – surtout de la rigueur pour comparer la page au modèle.

Exercice 2 – Maquette complexe avec CSS uniquement

Objectif : Compléter le fichier style.css d’un document existant afin d’obtenir exactement la mise en page d’une maquette (en-tête, colonnes, footer, images de fond).

Points clés : travailler avec plusieurs arrière‑plans (body, page, header, footer), gérer deux colonnes avec section et aside, positionner les éléments du header et utiliser correctement les chemins d’images (../img/...).

Difficulté ressentie : moyenne à élevée – nécessite de comprendre l’héritage des styles et l’impact de chaque propriété sur la mise en page.

Difficultés rencontrées

  • Retrouver les bonnes dimensions et marges pour que le contenu colle exactement à la maquette.
  • Comprendre le positionnement dans le header (texte centré, liens à droite) et gérer les espacements.
  • Gérer les colonnes section / aside avec des largeurs adaptées et des bordures ou séparateurs.
  • Corriger les chemins d’images dans le CSS (différence entre img/... et ../img/... selon l’emplacement du fichier).

Qualité du résultat obtenu

Les pages obtenues sont visuellement très proches des captures fournies : la structure générale, les couleurs et la typographie sont respectées.

Quelques petits écarts subsistent parfois au niveau des marges ou des espacements, mais l’ensemble correspond bien aux objectifs du TP.

Intérêt pour mon apprentissage

Ce TP est important car il pose les bases de tout le reste : savoir écrire un HTML propre et utiliser le CSS pour contrôler la présentation.

Il m’aide aussi à développer un œil plus précis sur les maquettes (alignements, hiérarchie visuelle, cohérence des styles) et à comprendre comment traduire un design en code.

Pistes d’amélioration

  • Commenter davantage le CSS pour mieux se souvenir du rôle de chaque bloc de styles.
  • Tester plusieurs navigateurs pour vérifier que l’affichage reste cohérent.
  • Commencer à organiser les fichiers (HTML, CSS, images) dans une structure de projet propre dès le début.
