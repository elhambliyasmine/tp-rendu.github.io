TP 2 – CSS Grid & Flexbox

Apprendre à construire des mises en page complexes avec CSS Grid Layout et Flexbox

Vue d’ensemble du TP

Le TP 2 est centré sur CSS Grid Layout et sur Flexbox, deux modules modernes pour créer des mises en page complexes en CSS.

Les exercices demandent de manipuler la grille avec et sans zones nommées, puis de reproduire les mêmes layouts en Flexbox, avant de réaliser une composition plus avancée d’éléments colorés numérotés.

Connaissances techniques mobilisées

  • Propriétés de base de CSS Grid : display: grid, grid-template-columns, grid-template-rows, gap.
  • Positionnement des éléments par lignes et colonnes avec grid-column et grid-row.
  • Utilisation des zones nommées avec grid-template-areas et grid-area pour rendre le code plus lisible.
  • Utilisation de Flexbox : display: flex, flex-direction, flex-grow, flex-basis, justify-content, align-items.
  • Traduction d’un même design en plusieurs techniques (Grid sans zones, Grid avec zones, puis Flexbox) pour comprendre leurs forces et limites.

Exercices réalisés

Exercice 1 – Grid Garden

Objectif : Compléter les 28 niveaux du jeu en ligne Grid Garden pour se familiariser avec la syntaxe de CSS Grid.

Points clés : pratique des lignes/colonnes, des propriétés grid-column-start, grid-column-end, gestion des fractions (fr) et du placement d’éléments dans la grille.

Exercice 2 – Maquette en 3 versions

Objectif : Reproduire la même page (header, sidebar, contenu principal, footer) de trois manières : Grid sans zones nommées, Grid avec zones nommées, puis Flexbox.

Points clés : comprendre la différence entre un positionnement “numérique” (lignes/colonnes) et un positionnement “sémantique” avec des zones nommées, puis réorganiser le HTML pour une version Flexbox lorsque Grid n’est pas utilisé.

Exercice 3 – Composition colorée

Objectif : Obtenir une grille de 9 blocs colorés qui respecte exactement la maquette fournie, en jouant sur les spans de lignes/colonnes (ou sur les tailles flex).

Points clés : maîtrise plus fine du placement, gestion des hauteurs/largeurs et respect de l’ordre HTML indiqué par les numéros.

Difficultés rencontrées

  • Bien visualiser la grille et comprendre quel bloc occupe quelles lignes/colonnes.
  • Se repérer entre les différentes syntaxes de Grid (lignes numériques, zones nommées) et ne pas se tromper dans les indices.
  • Adapter la structure HTML pour la version Flexbox tout en gardant le même rendu visuel.
  • Gérer les cas où plusieurs éléments doivent partager une même ligne ou une même colonne avec des proportions précises.

Qualité du résultat obtenu

Les pages reproduites avec Grid et Flexbox sont très proches des maquettes fournies, et les blocs colorés de l’exercice 3 suivent correctement l’agencement demandé.

Le code gagne en lisibilité dans la version avec zones nommées, et la version Flexbox montre qu’il est possible de construire des layouts complexes même sans Grid.

Intérêt pour mon apprentissage

Ce TP est essentiel pour comprendre comment organiser une page moderne avec des techniques puissantes comme CSS Grid et Flexbox.

Il aide aussi à choisir la bonne approche selon le type de layout : Grid pour des grilles 2D complexes, Flexbox pour des alignements et des lignes plus simples.

Pistes d’amélioration

  • Ajouter des media queries pour rendre les mises en page responsive sur mobile et tablette.
  • Refactoriser le CSS pour limiter les répétitions entre les trois versions de la maquette.
  • Expérimenter d’autres compositions en Grid en réutilisant les mêmes principes (zones nommées, spans, fractions).
