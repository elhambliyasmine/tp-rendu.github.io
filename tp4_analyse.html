TP 4 – Sélecteurs CSS avancés

Menus, attributs, pseudo‑éléments et tableaux stylés avec le minimum de HTML

Vue d’ensemble du TP

Le TP 4 est dédié aux sélecteurs CSS avancés et aux pseudo‑classes/pseudo‑éléments appliqués à des pages déjà codées en HTML.

L’idée est de modifier uniquement la feuille de style pour créer un menu interactif, des listes enrichies et un tableau mis en forme, sans toucher à la structure HTML d’origine.

Connaissances techniques mobilisées

  • Positionnement et gestion de blocs avec les propriétés de base (largeur fixe, marges, bordures, position fixe pour le menu).
  • Utilisation des sélecteurs d’attributs sur les liens (a[hreflang="en"] par exemple) pour cibler des éléments selon leurs attributs.
  • Usage des pseudo‑éléments ::before et ::after pour ajouter du contenu décoratif (icônes, drapeaux, puces personnalisées) sans modifier le HTML.
  • Utilisation des pseudo‑classes structurelles comme :nth-child() ou :nth-of-type() pour styliser les lignes ou colonnes d’un tableau.
  • Compréhension de la cascade et de la spécificité pour minimiser le nombre de règles CSS tout en couvrant tous les cas.

Exercices réalisés

Exercice 1 – Blocs + menu synchronisé

Objectif : Styliser trois blocs texte et un menu fixe à gauche de la page, puis faire disparaître les paragraphes du flux pour n’afficher que celui correspondant à l’item cliqué.

Points clés : largeur fixe de 600px, alignement des blocs à 250px du bord gauche, menu en position fixe, couleurs synchronisées entre item de menu, titre et bordure du paragraphe.

Exercice 2 – Liste de liens multi‑langues

Objectif : Reproduire une liste de liens décorés selon leur langue, uniquement avec le CSS (icônes, drapeaux, textes ajoutés).

Points clés : sélecteurs d’attributs sur hreflang, utilisation de ::before et ::after pour injecter des symboles, gestion fine des marges/paddings pour aligner les éléments.

Exercice 3 – Tableau stylé avec nth-child

Objectif : Mettre en forme un tableau HTML (lignes alternées, entêtes, surlignage de certaines colonnes) en minimisant le nombre de règles CSS.

Points clés : utilisation de :nth-child() ou :nth-of-type() pour cibler les lignes paires/impaires, colonnes spécifiques, survol des lignes, sans ajouter de classes dans le HTML.

Difficultés rencontrées

  • Bien comprendre comment fonctionne la position fixe pour que le menu reste en place pendant le scroll.
  • Écrire des sélecteurs d’attributs précis sans qu’ils se déclenchent sur les mauvais liens.
  • Maîtriser la syntaxe des pseudo‑éléments ::before / ::after et leur positionnement par rapport au texte.
  • Concevoir des règles :nth-child() génériques pour éviter de cibler chaque ligne du tableau une par une.

Qualité du résultat obtenu

Les pages obtenues respectent bien les captures fournies : le menu est fonctionnel, les liens sont enrichis visuellement et le tableau est lisible avec des lignes alternées et des colonnes mises en valeur.

Le code CSS reste relativement concis grâce à l’utilisation des sélecteurs avancés, ce qui facilite la maintenance et évite de polluer le HTML avec trop de classes.

Intérêt pour mon apprentissage

Ce TP montre la puissance des sélecteurs CSS : il est possible de transformer complètement l’apparence et le comportement d’une page sans modifier le code HTML.

Il prépare aussi à travailler sur des projets où le HTML est imposé (CMS, frameworks) et où il faut s’adapter en écrivant des règles CSS plus intelligentes.

Pistes d’amélioration

  • Commenter les sélecteurs complexes pour se rappeler à quoi ils servent.
  • Tester plusieurs navigateurs pour vérifier la compatibilité des pseudo‑classes utilisées.
  • Expérimenter d’autres effets avec ::before / ::after (icônes, animations simples, badges).
